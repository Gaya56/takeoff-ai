# CLAUDE.md — takeoff-ai Repo Breakdown

## What This Is
AI-powered construction takeoff tool. Upload a PDF floor plan → get back an annotated PDF with color-coded elements (walls, doors, windows, dimensions) + a material quantity report.

Client: Salem Al-Zahari, Genesis Open Developments Inc.

## Pipeline Flow (`pipeline/takeoff.py`)
1. `ingest_pdf(path)` — opens PDF with PyMuPDF, renders each page at 2x zoom to PNG
2. `analyze_plan(images)` — sends each page image to Gemini with structured JSON prompt
3. Gemini returns: legend (wall types), scale, detections (x1,y1,x2,y2 per element)
4. `annotate_pdf(output_path)` — draws colored lines on PDF using PyMuPDF shapes
5. `generate_report()` — aggregates linear footage per type per floor
6. `run(pdf_path, output_path)` — full pipeline, returns {annotated_pdf, report, flagged_count}

---

## What Works ✅
- Full pipeline end-to-end (PDF in → annotated PDF + JSON report out)
- Gemini reads legend and identifies wall type codes (M1, M2, C1, C2, C3 etc.)
- Color-coded annotations drawn directly on PDF
- Material quantity report (linear footage per wall type per floor)
- FastAPI backend with CORS enabled
- Frontend: drag-drop upload, results table, download button
- 3/5 test runs passed with real annotated PDFs generated (~3.2MB each)
- .gitignore excludes .venv (no bloat on GitHub)

## What Needs Work ⚠️
1. **Annotation precision** — lines follow walls but not pixel-perfect. Gemini coords are approximate.
2. **All elements** — prompt now covers doors/windows/dims/structural but needs real-world testing
3. **Legend-only pages** — page 0 of Salem's PDF is a legend page; sometimes gets annotated unnecessarily
4. **Speed** — ~4 min per 3-page PDF (3 sequential Gemini calls). Consider batching or parallelizing pages.
5. **Job persistence** — jobs stored in /tmp, wiped on restart. Needs Supabase or S3 for prod.
6. **No auth** — anyone with the URL can upload. Fine for MVP, needs API key or login for prod.
7. **Report format** — basic JSON/table. Salem wants Excel export eventually.
8. **Mobile UI** — works but not optimized for phone.

## Running Locally
```bash
git clone https://github.com/Gaya56/takeoff-ai
cd takeoff-ai
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
GEMINI_API_KEY=your_key uvicorn api.main:app --reload
# Open http://localhost:8000
```

Or use the helper:
```bash
GEMINI_API_KEY=your_key ./start.sh
```

## Running Tests
```bash
source .venv/bin/activate
GEMINI_API_KEY=your_key python tests/test_pipeline.py
# Outputs annotated PDFs to tests/output/
```

## Key Constants (pipeline/takeoff.py)
- `CONFIDENCE_THRESHOLD = 0.65` — below this, walls flagged for review (pink)
- `zoom = 2.0` — PDF render resolution multiplier (higher = better accuracy, slower)
- Model: `gemini-2.5-flash` — swap to `gemini-2.5-pro` for better accuracy (slower + costlier)
